Ataques y Protocolos de Red
Ataques de suplantación

Spoofing: Técnica basada en la suplantación de identidad técnica.
Spoofing de capa de red: Ataque enfocado específicamente en la manipulación del direccionamiento IP.

Protocolos y ataques en redes locales
802.1X: Protocolo para el control de acceso a la red basado en puertos.
Poisoning: Ataque de envenenamiento de la tabla MAC para redirigir tráfico.
Direccionamiento
Puertos: Rango máximo de 65,536.
IP Address: Dirección lógica utilizada para identificar dispositivos en la red.
MAC Address: Dirección física asociada al hardware.

Escaneo de servicios
Se revisaron puertos comunes detectados en auditorías de red, asociados a servicios como FTP, SSH, SMTP, DNS, HTTP, POP3, IMAP, bases de datos y acceso remoto.
Errores en sistemas de detección

Falso Positivo: El sistema detecta una amenaza inexistente.
Falso Negativo: El sistema no detecta una amenaza real.

Análisis de Problemas y Amenazas
Troubleshooting: Proceso sistemático para identificar la causa raíz de un problema técnico.
IP Spoofing: Suplantación de una dirección IP para engañar a un sistema.
Exploit: Código o conjunto de datos diseñado para aprovechar una vulnerabilidad específica.
NVD: Base de datos oficial que centraliza información sobre vulnerabilidades conocidas.

Frameworks de análisis de amenazas
STRIDE: Modelo para identificar amenazas durante el diseño de software.
MITRE ATT&CK: Base de conocimiento sobre tácticas y técnicas usadas por atacantes reales.


Tipos de atacantes
Un hacker se define como alguien capaz de hacer que un sistema funcione de una manera distinta a la diseñada originalmente.
Black Hat: Actúan con fines maliciosos.
White Hat: Hackers éticos que buscan mejorar la seguridad.
Grey Hat: Perfil intermedio, actúan sin permiso pero suelen reportar fallos.
Hacktivistas: Motivados por causas ideológicas o políticas.
Suicidal Hackers: No les importa ser identificados o capturados.
Script Kiddies (Plug and Play): Usan herramientas sin comprender su funcionamiento.

Vulnerabilidades y Fases de Ataque
Conceptos clave
Payload: Código o datos enviados para explotar una vulnerabilidad.
Zero Day: Vulnerabilidad explotada antes de que exista una solución.
Flaw: Error o falla técnica en un sistema.
Log4j: Biblioteca de Java afectada por una vulnerabilidad crítica.
vsftpd 2.3.4: Ejemplo histórico de software con una puerta trasera.
Shodan: Motor de búsqueda de dispositivos conectados a Internet.

Fases de un ataque
Reconnaissance: Reconocimiento inicial del objetivo.
Footprinting: Recolección de información pública y técnica.
Network Mapping: Mapeo de activos y servicios de red.
Gaining Access: Explotación para obtener acceso.
Daisy Chaining / Covering Tracks: Mantener acceso y ocultar evidencias.
Centros de operación

SIEM: Centraliza y analiza registros de seguridad.
SOC: Centro dedicado a la ciberseguridad.
NOC: Centro enfocado en la operación y rendimiento de la red.

Factor Humano y Criptografía
Ingeniería Social
Conjunto de técnicas de manipulación psicológica para engañar a personas y obtener información o acceso. Se identificó al ser humano como el eslabón más débil de la seguridad.

Tipos de ataques:
Phishing: Ataque masivo de suplantación.
Spear Phishing: Ataque dirigido y personalizado.
Whaling: Ataque enfocado en altos directivos.
Criptografía y algoritmos
Cifrado César: Método de sustitución que desplaza letras del alfabeto.
Cifrado: Protege información mediante una clave.
Codificación: Cambia el formato de datos para transmisión.
Hashing: Genera una huella única e irreversible.
Entropía de contraseñas: Mide la dificultad de romper una contraseña por fuerza bruta.
